# Markdown → HWPX 역변환 스킬 추가 (kordoc 2.5.1 + lk-doc-md2hwpx)

**일시**: 2026-04-22 12:50 (KST)
**관련 이슈**: [chrisryugj/kordoc#4](https://github.com/chrisryugj/aleod/issues/4) — HWPX 생성기가 한글에서 안 열리는 문제
**플러그인**: leeloo-doc 1.1.1 → 1.2.0

## 지시 요약

이전에 시도했다가 멈췄던 kordoc의 Markdown → HWPX 변환 테스트를 재개. upstream 후속 수정(cd85cb6 + v2.5.0)이 적용된 버전을 검증하고, 사내에서 활용 가능하면 새 skill로 추가.

## 작업 흐름

### 1단계 — cd85cb6(v2.4.1) 검증 (오전)

`/tmp/kordoc-md2hwpx`에 격리 환경 구성, kordoc 2.4.1 설치 → 다양한 markdown 요소(헤딩/굵게/기울임/코드/인용/리스트/테이블)를 한 파일에 담아 변환.

생성 HWPX 구조 확인:
- `META-INF/container.xml` OCF 네임스페이스 적용 ✅
- `Contents/header.xml` 신규 추가 (2011 head/paragraph 네임스페이스, 7개 언어 폰트) ✅
- `Contents/section0.xml` 2011 네임스페이스 + paraPrIDRef/charPrIDRef ✅

macOS 한글에서 **파일이 열림** (이전 PR #9 시점에는 거부됐었음). 다만:
- 테이블 누락 (제목만 있고 표 본체 사라짐)
- 인라인 서식 시각 구분 없음
- 순서 있는 목록 자동 번호 미작동

이슈 #4에 첫 follow-up 댓글 게시. owner가 1시간 만에 v2.5.0/v2.5.1로 테이블 풀 스펙 수정 응답.

### 2단계 — v2.5.1 재검증 (오전 후반)

같은 sample.md로 재변환. section0.xml의 `<hp:tbl>`이 **풀 스펙**으로 변경됨:
- 속성 10종 (`rowCnt="4"`, `colCnt="3"`, `cellSpacing`, `borderFillIDRef="1"`, `noShading`, ...)
- `<hp:sz>`, `<hp:pos treatAsChar="1">`, `<hp:outMargin>`, `<hp:inMargin>`
- 셀 내부 `<hp:subList>` 래퍼 + `<hp:cellAddr>` + `<hp:cellSpan>` + `<hp:cellSz>` + `<hp:cellMargin>`
- `<hp:tbl>`을 `<hp:p><hp:run>` 안에 paragraph-anchored로 배치
- `borderFills itemCnt="2"` (id=1 SOLID 0.12mm 정의)
- `Preview/PrvText.txt` 추가 (macOS 한컴이 확인하는 경로)

한글에서 다시 열기 → **표 4행×3열 정상 렌더링** 🎉.

다만 테두리 라인이 화면에 안 보임. header.xml 분석 결과 단위 표기 불일치 발견:
- `footNotePr/noteLine`: `width="0.12 mm"` (공백 있음 — 한글이 직접 만든 표기)
- `borderFill/*Border`: `width="0.12mm"` (공백 없음)

#### 현실 비유 — 인테리어 도면의 단위 표기

실측을 한 인테리어 도면에서 한 곳은 "1200 mm", 다른 곳은 "1200mm"로 표기되어 있다고 하자. 사람은 둘 다 같은 1.2m로 읽지만, 자동 발주 시스템이 정규식 `(\d+)\s+mm`로만 파싱한다면 "1200mm"는 매칭 실패해 무시된다. HWPX 파서도 동일하게 한 가지 표기만 인식하도록 구현되어 있을 가능성이 큼 → 결과적으로 borderFill type이 SOLID로 정의돼 있어도 한글이 NONE으로 fallback해 테두리가 그려지지 않음.

### 3단계 — leeloo-doc 통합

격리 검증이 통과해 사내 플러그인에 흡수:

1. `leeloo-doc/package.json`: `kordoc ^1.9.0` → `^2.5.1` (메이저 점프 2단계)
2. 기존 래퍼 호환 확인 — `parse`, `compare`, `extractFormFields` 모두 v2.5.1에서도 동일 시그니처
3. 새 스크립트 `scripts/kordoc-md2hwpx.mjs` — `markdownToHwpx()` API 1줄 호출 + 입력 검증
4. 새 SKILL.md `skills/lk-doc-md2hwpx/SKILL.md` — 인자/Phase 0~2 + **알려진 제약 3건 명시** (테두리/인라인 서식/ordered list)
5. `plugin.json` 1.1.1 → 1.2.0, `marketplace.json` 동기화
6. `leeloo-doc/CLAUDE.md` (scripts 4→5, skills 4→5) 및 root `CLAUDE.md` 동기화

스모크 테스트 결과: 31,453 bytes / 5ms / 9 files (v2.5.1 ZIP 구조와 일치).

```javascript
import { readFile, writeFile, stat } from "node:fs/promises";
import { resolve, dirname, basename, extname, join } from "node:path";
import { markdownToHwpx } from "kordoc";

const [,, inputArg, outputArg] = process.argv;
// ... (인자 검증, 출력 경로 추론)
const md = await readFile(inputPath, "utf8");

await writeFile(outputPath, Buffer.from(buf));
```

## 후속 작업

- kordoc upstream에서 단위 표기 1줄 패치(`"0.12mm"` → `"0.12 mm"`)가 들어오면 테이블 테두리 표시 가능. 별도 PR 제안 가치 있음.
- 인라인 서식·ordered list는 `<hp:list>` 구조 + `fontRef boldFontRef` 설계 변경이 필요해 보임 — 구조적 작업이므로 시간 두고 지켜볼 것.
