# lk-commit 커밋 메시지 표준 적용

## 지시 요약
commit-messages-guide 표준 문서(RomuloOliveira/commit-messages-guide)를 토대로 lk-commit 스킬의 커밋 메시지 생성 규칙 재정의.

## 작업 내용
- Haiku 프롬프트 템플릿의 규칙 섹션을 표준 기반으로 전면 교체
- 2부분(제목+본문) → 3부분(제목+본문+footer) 구조로 확장
- 제목 규칙: 50자 이내, 과거형 금지, 코드 없이 이해 가능하게 작성
- 본문 규칙: 72자/줄, '왜' 변경했는지 맥락 설명, 부수 효과 명시
- footer: Refs/Closes 이슈 참조 지원 추가
- 금지 규칙 명시: 총칭적 메시지, 맥락 없는 파일명 나열
- 예시 2개 → 3개 (단순/표준/복합)로 확장
