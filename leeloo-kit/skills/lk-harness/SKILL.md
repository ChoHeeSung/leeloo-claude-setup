# /lk-harness — Harness Engineering Audit

Audits the health of the leeloo-kit harness and prints detailed reports for violations. Since the Stop hook only outputs a one-line summary, use this skill for detailed investigation.

## Subcommands

```
/lk-harness                         — same as context-lint (default)
/lk-harness context-lint            — context budget audit summary
/lk-harness context-lint --verbose  — detailed violations
/lk-harness budget                  — today + 7-day token estimate summary
/lk-harness budget --week           — 7-day daily table
/lk-harness budget --top-skills     — 14-day skill usage ranking
/lk-harness budget --load           — current auto-loaded context status
/lk-harness failure-memory          — counts per type + top patterns summary
/lk-harness failure-memory --force  — bypass once-per-day gate and rotate immediately
/lk-harness cache-audit             — CLAUDE.md prefix volatility summary
/lk-harness cache-audit --verbose   — per-file block structure
/lk-harness cache-audit --file <p>  — single-file detail
```

> All audit subcommands live **only in this skill**. Do not scatter them across other skills (SRP).

## Procedure

### Argument Parsing

Parse the subcommand from user input.

- No argument or `context-lint` → **context-lint** action
- `budget` → **budget** action
- `failure-memory` → **failure-memory** action (rotate + summary)
- `cache-audit` → **cache-audit** action
- Flag `--verbose` → context-lint / cache-audit detail
- Flags `--week` / `--top-skills` / `--load` → budget view selection
- Flag `--force` → bypass failure-memory once-per-day gate
- Flag `--file <path>` → cache-audit single file

### context-lint action

Audits the harness context budget.

**Audit targets**

| Category | Target | Default Threshold |
|----------|--------|-------------------|
| skill-description | `*/skills/*/SKILL.md` frontmatter `description` length | 100 chars |
| skill-argument-hint | same frontmatter `argument-hint` length | 120 chars |
| claude-md-root | root `CLAUDE.md` line count | 60 lines |
| claude-md-plugin | each plugin `CLAUDE.md` line count | 70 lines |
| commands-drift | `commands/*.md` ↔ `SKILL.md` frontmatter alignment | exact match |

**Execution**

Default (summary):
```bash
node leeloo-kit/scripts/context-lint.js
```

Detail (--verbose):
```bash
node leeloo-kit/scripts/context-lint.js --verbose
```

Sample output (detail):

```
context-lint: 2건
  [skill-description] leeloo-doc/skills/lk-doc-parse/SKILL.md — 142 > 100
  [commands-drift] leeloo-kit/commands/lk-persona.md (out-of-sync)
```

**Threshold tuning**

Defaults live in `leeloo-kit/resources/context-budget.default.json` (git-tracked). Per-project overrides go in `.leeloo/context-budget.json` with the same keys (shallow merge).

```json
{
  "skill_description_max": 100,
  "skill_argument_hint_max": 120,
  "claude_md_plugin_max_lines": 70,
  "claude_md_root_max_lines": 60,
  "hook_output_max_chars": 300,
  "commands_drift_check": true
}
```

**Stop-hook auto-audit**

The same checks run automatically on session end (Stop event), printing only a one-line summary on violation. Use this command (`--verbose`) for details. Auto-execution is silent-fail and never blocks the existing Stop behavior.

**Drift auto-fix**

`commands-drift` violations are immediately regenerated by:

```bash
node leeloo-kit/scripts/generate-commands.js --sync
```

Always run this after changing a SKILL.md `description` or `argument-hint`.

### budget action

Aggregates per-session auto-load estimates and skill execution frequency. Raw data is silent-fail-accumulated by hooks (SessionStart/SessionEnd/skill-post) into `.leeloo/token-budget/<YYYY-MM-DD>.jsonl`.

**Data Properties**

| Item | Method | Accuracy |
|------|--------|----------|
| Auto-load chars | sum of root CLAUDE.md + plugin CLAUDE.md + all SKILL.md frontmatter file sizes | ±5% |
| tokens_est | `chars / 3.5` (mixed EN/KR average) | ±15% |
| Skill usage frequency | PostToolUse:Skill event count | exact |

**For trend** observation. Absolute values may differ from Anthropic's actual billing; always compare relatively (e.g., week-over-week).

**Execution**

```bash
# Today + 7-day summary (default)
node leeloo-kit/scripts/budget-report.js

# 7-day daily table
node leeloo-kit/scripts/budget-report.js --week

# 14-day skill usage ranking
node leeloo-kit/scripts/budget-report.js --top-skills

# Current auto-loaded context status
node leeloo-kit/scripts/budget-report.js --load
```

**Statusline display**

`leeloo-kit/resources/statusline-leeloo.sh` displays today's auto-load average (`X.XK` tok) in a purple block. Install/update via `/lk-setup statusline` or `cp leeloo-kit/resources/statusline-leeloo.sh ~/.claude/`.

**Retention policy**

- `.leeloo/token-budget/<YYYY-MM-DD>.jsonl` — kept for 30 days
- Files older than 30 days are auto-merged into `.leeloo/token-budget/archive/<YYYY-MM>.jsonl` (handled by SessionStart/SessionEnd hooks)

**tokens_per_char tuning**

To adjust the estimation coefficient based on your project's character distribution, add a `tokens_per_char` field to `.leeloo/context-budget.json` (default `0.2857` ≈ 1/3.5). Currently the token-budget module default is used.

### failure-memory action

**Ages and clusters** the `.leeloo/failure-memory/<type>.md` records.

**Procedure**

1. Sort each type's records (`<type>.md`) by date
2. Move records beyond `KEEP_RECENT=50` into `.leeloo/failure-memory/archive/<type>-<YYYY-MM>.md`
3. Normalize the remaining error bodies (mask absolute paths, hashes, timestamps, Edit JSON payloads), then frequency-cluster
4. Replace the `## Failure Memory` section in the project-local `CLAUDE.local.md` with per-type summaries
   - Per-type records **≥ 5**: render top-3 pattern clusters
   - Per-type records **< 5**: dump originals in reverse chronological order (payload masked) — keep originals when statistical confidence is low

**Execution**

```bash
# Auto-runs once per day (SessionEnd). Manual rerun:
node leeloo-kit/scripts/failure-memory-rotate.js --force
```

**Normalization rules**

| Source | Replacement |
|---|---|
| Edit tool failure JSON payload (`{"filePath":...` ~ end of line) | `{payload}` |
| Absolute path (`/Users/...`, `/home/...`) | `{path}` |
| File path (`foo/bar.js`) | `{path}` |
| Hex hash (≥ 8 chars) | `{hash}` |
| Timestamp (ISO) | `{ts}` |
| `line \d+` | `line {n}` |

The first 80 normalized characters serve as the cluster key. JSON payload masking applies to `< 5` raw dumps as well, removing noise from the summary section.

**Gate behavior**

- Records the last execution date (KST) in `.leeloo/.last-rotate`
- Reruns on the same day return silently with `skipped: gated-today`
- `--force` bypasses the gate

**Archive restore**

```bash
cat .leeloo/failure-memory/archive/<type>-<YYYY-MM>.md >> .leeloo/failure-memory/<type>.md
```

### cache-audit action

Audits the **prefix stability** of CLAUDE.md. Prompt cache assumes a stable prefix to hit, so frequently changing blocks in the top 50% trigger cache misses.

**Formula**

```
volatility = file commit count over last 30 days / file line count
```

| Score | Interpretation |
|---|---|
| ≥ 0.5 | 🔴 Highly unstable — prioritize relocation |
| ≥ 0.2 | 🟡 Caution — review top blocks |
| < 0.2 | ✓ Stable |

**Target files**

- root `CLAUDE.md`
- each plugin's `<plugin>/CLAUDE.md`

**Execution**

```bash
# Summary
node leeloo-kit/scripts/cache-audit.js

# Per-file block structure
node leeloo-kit/scripts/cache-audit.js --verbose

# Single file
node leeloo-kit/scripts/cache-audit.js --file CLAUDE.md

# Ignore cache (24h TTL)
node leeloo-kit/scripts/cache-audit.js --no-cache
```

**Relocation guidance**

Move blocks that change every session (e.g., `## Failure Memory`) to the **bottom** of the file. Place stable overviews, architecture, and command tables on top. The audit tool only recommends — it does not auto-relocate.

**Cache**

- `.leeloo/cache-audit.cache.json` — 24-hour TTL
- Reduces git log cost; bypass with `--no-cache`

## Notes

- This skill is **read-only audit**. It does not auto-modify files.
- To allow violations, adjust thresholds or add paths to `ignore_paths` in `.leeloo/context-budget.json`.
- `context-lint.js` works as both CLI and module; new Tier 2/3 audits (`token-budget`, `cache-audit`, `failure-memory-rotate`) will accumulate in parallel under the same pattern.
